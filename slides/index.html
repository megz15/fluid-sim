Governing Equations

Incompressible Navier--Stokes

∇·u = 0     // mass conservation
∂u/∂t + (u·∇)u = −(1/ρ)∇p + ν∇²u + f

Re = UD/ν — single parameter controlling flow regime.
At Re = 400: periodic vortex shedding regime for a circular cylinder.

ω = ∂v/∂x − ∂u/∂y   // 2D vorticity
St = fD/U   // Strouhal number

Methodology I

Eulerian FDM — Chorin's Projection

MAC Staggered Grid

Scalar quantities (p, ρ) at cell centres; u on vertical faces, v on horizontal faces.
Ghost layer: domain is (Nx+2)×(Ny+2). Prevents pressure--velocity checkerboard instability.

Fractional Step (5 stages)

  1. Body forces (Euler ODE)
  2. Viscous diffusion — implicit Gauss--Seidel
  3. Pressure projection — SOR (ω=1.9, 40 iters)
  4. Boundary conditions (Neumann or Dirichlet)
  5. Semi-Lagrangian advection (unconditionally stable)
Methodology I

FDM — Update Equations

Diffusion update

un+1i,j = (uni,j + a·Σ neighbours) / (1 + 4a)

Pressure correction

p_corr = −div / s₀  ·  ω_SOR

Semi-Lagrangian advection

un+1(x) = un(x − unΔt)
Methodology II

Lattice Boltzmann Method — D2Q9 BGK

Kinetic Theory Basis

Evolves discrete probability distributions fk(x,t).
Macroscopic fields recovered as moments:

ρ = Σ fk  ,  ρu = Σ fk ek
Methodology II

BGK Collision

fk(x,t+1) = fk − (1/τ)(fk − feqk)
feqk = wkρ[1 + 3(e·u) + 4.5(e·u)² − 1.5|u|²]

Chapman--Enskog:  ν = (τ − 0.5)/3
Stability requires τ > 0.5

Methodology II

LBM — Lattice & Boundary Conditions

D2Q9 Lattice

9 velocity directions; c²s = 1/3 (lattice units)
Weights: w₀=4/9, w₁₋₄=1/9, w₅₋₈=1/36

Boundary Conditions

  • Solid obstacles: bounce-back (no-slip)
  • Inlet: equilibrium at Uin, ρ=1; velocity ramp over Nramp=500 steps
  • Outlet: zero-gradient: fk(N-1) ← fk(N-2)
  • Top/bottom: bounce-back (no-slip) or specular flip (free-slip)

Key difference from FDM

No global pressure solve. Collision & streaming are perfectly local and parallel.

Software Architecture

Zero-Dependency Implementation

Native Engine

The interactive web application is entirely self-developed. Both 2D simulations are written from scratch in TypeScript and run entirely in the browser via SvelteKit without reliance on third-party physics or CFD libraries.

No native code, WebAssembly, or server-side computation is used. All field calculations and pixel manipulations are computed natively and painted directly to the HTML5 Canvas via the ImageData API.

Extension I

Wake Diagnostics — Strouhal Number

Strouhal Number Probe

Probe at (x₁, yprobe) = (cx + r·d₁, cy + 0.5r), offset above centreline. Detects rising zero-crossings of v(t):

vn-1 ≤ 0 and vn > 0 ⟹ crossing at n

Period T = rolling average of last 5 inter-crossing intervals.

St = D / (T · U)

Expected St at Re = 400

Cylinder: ≈ 0.20--0.21 (Norberg 1994).
Simulation converged to St ≈ 0.12 for both methods — lower due to coarser grid & 2D limitation.

Extension I

Wake Diagnostics — Karman Spacing

Bearman Two-Probe Method

Second probe at x₂ = cx + r·d₂. Time delay Δt = Δx/Uc between crossings gives convection velocity.

0.4U < Uc = Δx/Δt ≤ 1.1U

Longitudinal spacing: a = Uc·T
Transverse spacing b from peak vorticity rows:

b = ⟨|ymax − ymin|⟩cycle · h
Extension I

Karman Spacing Ratio

Displayed as b/a in real-time. Highlighted green when within 0.1 of theoretical stability criterion:

b/a = (1/π) arcsinh(1) ≈ 0.281
Extension II

Vorticity Visualisation Mode

Added as an alternative to velocity-magnitude rendering. Computed via centred differences:

ωi,j = (vi+1,j − vi-1,j)/(2h) − (ui,j+1 − ui,j-1)/(2h)

Mapped to a diverging colourmap: ■ Blue = clockwise (ω < 0) · ■ White = zero · ■ Red = counter-clockwise (ω > 0)

Extension II

Vorticity Visualisation — Advantages

Advantages over speed-magnitude field:

  • Suppresses uniform background flow → vortex cores are immediately visible
  • Directly identifies shedding asymmetry
  • Used to locate ymax/ymin vortex rows for b/a measurement
  • Obstacle offset (cy = 35 vs. Ny/2 = 40) visible — ensures deterministic first shed

Discussion

Comparison of Methods

Where methods agree

  • Both reproduce von Karman vortex street for all 7 shapes
  • Same qualitative wake structure at matched Re = 400
  • Consistent Strouhal numbers

Sources of discrepancy

  • LBM compressibility error: Recovers incompressible NS only up to O(Ma2). With Ma ≈ 0.35, small density fluctuations bias St upwards.
  • FDM numerical dissipation: Semi-Lagrangian advection smooths gradients. LBM's streaming step is exact.
  • Karman b/a convergence: Finite domain truncates street before asymptotic 0.281.
  • LBM startup: Inlet ramp (500 steps) gives a smoother startup. FDM steps full U immediately.
Discussion

3D Results at Re ≈ 400

3D at Re ≈ 400

At Re = 400, a circular cylinder is within the transitional regime where three-dimensional instabilities are known to develop. The 3D D3Q19 solver captures this physical spanwise deformation that is structurally missed by 2D simulations.
